# --- the sprite/level-bank loader and codec (Part IV §3 tile map, §4 obstacles) ---
0080B4  ilb_load         load an .ilb/.vlb sprite bank for the current course: index a buffer table, read+unpack the WHOLE file (bank_read_unpack — one ByteRun1 stream), then walk the bank's descriptors IN the unpacked buffer at buf+2, stride $14=20. Each 20-byte descriptor: +0 type byte (1=stored cell, 0=composited), +1 flag byte (bit0 = process), +2 width in 16-px words, +4 height in rows, +6 one-plane byte size (=w*2*h), +8 source offset (relocated +buf), +$C dest cell ptr (allocated from buffer slack or relocated), +$10 aux ptr (relocated). Cells' source pixel data is contiguous after the descriptor array; plane count = source span / +6 (2 for small objects, 4 for scenery). Composites type-0 cells via composite_planes.
007F38  mlb_load         load an .mlb level module (different container: flag $01 + a 5-entry offset-table header) through the SAME bank_read_unpack path, so it shares the ByteRun1 codec.
007E4E  bank_read_unpack file loader: dos_open(name, MODE_OLDFILE) -> dos_read into the buffer -> dos_close; then, if the global flag $8351 is set, move the loaded (packed) data to the high end of the buffer and ByteRun1-decompress it in place (unpack_byterun1) back to the start. Returns the unpacked length.
007DA6  pool_alloc       bump allocator: hand out $8(a6) bytes from the work pool tracked by $8340/$8344 (high-water at $8334), growing it via $8D74 when exhausted.
009118  unpack_byterun1  THE BITMAP CODEC = ByteRun1 / PackBits (identical to IFF ILBM bodies). unpack(src@8, dst@C, srclen@10, dstlen@14): read a signed control byte n — n in 0..127 copies the next n+1 bytes literally; n in -1..-127 repeats the next single byte (1-n) times (2..128); n = -128 is a no-op. Stops at dstlen or end of source.
008026  composite_planes one bank cell: zero the dest cell (size = desc->4 * desc->2 * 2), then OR four consecutive planar source blocks (desc->8) into it — builds the final 2-plane (16-wide) cell from the unpacked source.

# --- marble physics & controls (Part V §4) ---
# PROVEN velocity facts: the marble (player 1) is object $236 (player 2 $32A); its
# velocity is obj+0/+4/+8 and position obj+$C/+$10/+$14 (object_draw $14EF0 at
# $14FFC integrates pos += velocity).
013E9A  marble_state_machine  THE marble STATE MACHINE (a5=marble). d7 = obj+$1A = the
#   marble state, 0..11 (12 states). Prologue $13EAC: for most states runs a wobble/settle
#   anim (obj+$EA gate, obj+$72/$73 bob counter+dir, obj+$7A timer); states 2,4,7,A,B skip
#   it. $13F2C: d0=state; CMPI.l #$C; BCC $14486 (default = state>=12); ASL #2; JMP
#   $2(pc,d0.l) -> 12-entry table at $13F40. obj+$1A doubles as the per-surface max-speed
#   selector read by $14E7E ($14D28). The 12 handlers (●=player-controllable: input $12F8C
#   + physics $14EF0; others redraw at current pos via $E74C, no input):
#     s0 $141DC ● ROLLING (normal play); if stun flag +$DD set -> state 8.
#     s1 $1427C ● LANDING: marble drops off a ledge -> airborne (+$36=2, in the update
#                 $14EF0); on touchdown $14884 sets s1 with the fall height -> impact-scaled
#                 bounce anim (+$60 eased to +$61), -> 0.
#     s2 $1436E   animation frame only (advances anim cursor +$64).
#     s3 $13FBA   EDGE/LANDING reaction: plays the sound for terrain +$62, clears, -> 0
#                 (entered by the wall/edge handlers $16xx/$17xx).
#     s4 $14072   FALLING/SETTLING: collision-query $15FC8; on contact runs appear-anim +
#                 latch region $14706, -> 0.
#     s5 $142FC ● OBJECT BUMP: the collision scan $1A86C (Track actor list $1ABE0, only
#                 courses $5D6 in {1,2,5}) drops the marble here on contact -> flat +$60
#                 countdown, -> 0.
#     s6 $13F70 ● COURSE INTRO entrance run (entered at course_init $3380); +$61 down, freeze vel.
#     s7 $1435E   transient: save-state + clear +$1C.
#     s8 $143FC   DIZZY (stunned): plays the swirl anim +$DE, tallies +$74, -> 0. Entered
#                 ONLY from s0 on a hard hit / survivable fall. RECOVERABLE, NOT death.
#     s9 $1439A   SPAWN/appear (entered from the spawn routine $14804).
#     s10 $14100  HOLE/REGION CAPTURE: runs the active region's script $FD68 + sound, -> s4
#                 (entered from the slope handlers $17AFA/$17CDA).
#     s11 $143C8  settle onto a scripted region (like s4, latches on contact).
# THE STATIC-SLOPE FORCE (the long-open path: how the $9A6 height MESH rolls the marble):
#   NOT sub_016900 (that is the scripted dynamic regions). It is the HEIGHT-FIELD GRADIENT,
#   read each frame by object_draw -> $EA10 then applied by $14A88.
00EA10  surface_sample   read the marble's 2x2 mesh cells (the four 8-byte corner-height
#   blocks $1EBA/$1EC2/$1ECA/$1ED2, filled from the $CCA mesh) and interpolate. $6D6 (iso
#   diamond half) picks which TRIANGLE of the cell the marble is in; $6D8/$6DA = the surface
#   GRADIENT (corner-height differences along x/y); then height = $6D8*$6D2 + $6DA*$6D4
#   (sub-cell), <<$D, + cornerZ<<$10 = the interpolated surface Z. So the triangular slope
#   faces are real: the gradient is taken from one of the two triangles of the tile quad.
00EF90  surface_sides    classify the 4 sides of the marble's cell from neighbour corner
#   heights ($1EBA..$1ED2): set side TYPE flags $6A4-$6A7 (0/1/3 = none/edge/wall) and side
#   values $6A8/$6AA/$6AC/$6AE. Feeds edge collision $EB64.
00EB64  edge_collision   per-side wall/edge response: when the marble's sub-cell pos
#   ($6D2/$6D4) crosses a boundary and that side's flag ($6A4-$6A7) is a wall, clamp the
#   velocity component ($EB2C/$EB48 adjust obj+0/+4).
014A88  apply_slope_force  THE static-slope downhill force (a5=marble). Read gradient
#   $6D8/$6DA (from surface_sample $EA10), x4 if steep (|g|>=$C), then marble velocity
#   -= gradient<<$B:  $14AEA SUB.l d0,(a5) vX -= gX<<11 ; $14AF6 SUB.l d0,$4(a5) vY -= gY<<11
#   = accelerate DOWNHILL (down the height gradient). Course 4 (aerial $5D6==4) ADDs instead
#   = inverted gravity. Called from the rolling-state handler at $0156DC.
012F8C  marble_input     INPUT -> velocity. For the marble ($236) reads the trackball
#   force ($195EA/$195EE; player 2 $195EC/$195F0), NEGates X, scales (*$160, >>8, * a
#   per-state factor from obj+$60/$1A, <<3), and ADDS it to the velocity:
#   $0130DA ADD.l d0,(a0)  -> obj+0 (vY) += force ;  $0130E0 ADD.l d0,$4(a0) -> obj+4
#   (vX) += force. (Then JSRs $14E7E.) PROVEN, no assumptions.
00DEBC  build_surface    $9A6 course descriptor -> $CCA surface-region structs. Reads
#   the descriptor (a0=$9A6): +$08 command/coordinate stream (bit selector + [col,row]
#   +value words, $FFFE/$FFFF terminated), +$18 world height, +$1A region count, +$1C
#   region-record table, +$20 edge-run sub-shape ptr table, +$24 width, +$26 per-column
#   word array (->$9AE/$9B2). Clears the $CCA buffer ($420 longs), then for each region
#   calls sub_00E158 to rasterise an 8-byte region record into the $CCA structs. PROVEN:
#   $9A6 terrain -> $CCA region structs.
00E158  build_region     rasterise one 8-byte region record into the $CCA CORNER-HEIGHT
#   mesh (a 22-wide grid of 8-byte cells, each = 4 corner heights at +0/+2/+4/+6). Record
#   = [+0 origin.x(s8)][+1 origin.y][+2 size.x][+3 size.y][+4..5 baseHeight word][+6 low5 =
#   edge-shape selector into the $9A6+$20 table -> a height-PROFILE byte array][+7 low3 =
#   SLOPE DIRECTION 0..7, indexes $2504 for the step (dx,dy); bit3 = flip]. $2504 holds
#   only the 4 iso diagonals (+/-1,+/-1); dir picks 1 of 8 fills (4 diagonals x 2 loop-
#   transpose orders, dir<4 vs dir>=4 take the two inner loops $E274/$E464). Walks the
#   rectangle DIAGONALLY in iso space: per step cell row=xAcc+yAcc-originX, col=originY-
#   xAcc+(row>>1), addr=$CCA+(row*$16+col)*8 (odd rows clamp to 21 cols = iso half-cell).
#   Writes value = baseHeight +/- profile[i] (flip = sign), profile advanced 1 byte/cell
#   and RESTARTED on a $80 marker (so 00 02 04..80 = repeating linear ramp) -> the height
#   RAMPS along the diagonal = the slope. A boundary test writes value into the right
#   corner word(s) of the cell depending on left/right (x0/xEnd) and top/bottom (y0/yEnd)
#   edges, sharing corners so adjacent cells agree = a continuous corner-height terrain.
#   NOTE $CCA is a MULTI-PHASE work buffer (verified same literal $0CCA at 3 strides):
#   build_region writes 8-byte corner-height cells; region_clear $8F42 inits 86-byte
#   region structs; particle/effect system $1E21C reuses 8-byte pos+vel records.
008F42  region_clear     init all 25 $CCA region structs (stride $56): +$18=0 (invalid),
#   +$C=0 / +$10=0 (LONG ref point), +$1F=0 (terrain code), +$19=index(0..24). So the ref
#   point starts at ZERO and is filled later by region_script $FD68 (NOT a static field).
00F7EA  region_build_list  populate the 86-byte $CCA region structs from a per-course
#   source list (head = *($FD2C)/Track header+0). Walks [x][y][scriptPtr-long] records; per
#   entry (filtered by tile-range args) allocs a struct via $F6B2, sets +$52/+$54 = (x,y)
#   grid key, and calls region_update $F96A which stores the record's long as the region's
#   SCRIPT POINTER +$3A and sets +$18/+$1A. region_activate $F8FC matches +$52/+$54 vs the
#   marble tile each frame to set +$18 valid.
00FD68  region_script    PER-REGION BYTECODE INTERPRETER (a5=region) — THIS writes the
#   downhill reference +$C/+$10. +$3A=script base, +$36=script PC (advances 2B/word read),
#   +$1E=current keyframe DURATION in frames (counted down; at 1 fetch next opcode). Word
#   opcode 0..$12 -> 19-entry jump table $FD96 (target=entry+2+off). KEY OPS:
#   op0 KEYFRAME ($FDE2): read 5 words -> +$C=refX(word<<$13/19), +$10=refY(word<<19),
#     +$14=refZ(word<<$10/16), +$1E=duration, +$1F=terrain code(5..59). Plants the ref pt.
#   op16 MOVE ($1002E): each frame +$C += region.vX(+0), +$10 += region.vY(+4) -> the ref
#     point DRIFTS = a MOVING slope/seesaw/sliding wall.
#   op2 ($1007A): set +$1C/+$23/+$1A=2 + sound = state-change keyframe.
#   So a scripted region's +$C/+$10 = a KEYFRAME STREAM (from the Track anim-script table
#   $FD2C). Fixed-point <<19 matches the marble position scale so sub_016900 subtracts
#   directly. OPEN: full 19-opcode vocabulary.
# DATA CHECK (PrcTrack decoded): the practice course has TWO terrain structures:
#   (1) $9A6 descriptor (Track header+0) = 66 STATIC slope records (8B each: 7x7 blocks,
#       dir 0/4 = the checkerboard diagonals, descending baseHeight) -> build_region mesh.
#       NO script, NO animation. THESE are the checkerboard slopes the player rolls down.
#   (2) Track header+$14 anim-script list = only 13 DYNAMIC regions, codes 5/17/26-31
#       (triggers / holes / start-finish / ball-catcher), NONE slope codes 11/13.
#   So region_script $FD68 drives only the dynamic regions; static slopes are NOT scripted.
#   OPEN (re-trace needed): how the 66-record corner-height MESH is sampled into the per-
#   frame slope force (sub_016900's code-11/13 path is fed by region +$C/+$10, not the
#   mesh; codes 11/13 never appear in the practice scripts).
00F8FC  region_activate  per-frame pass over the 25 $CCA region structs (a4=$CCA, +=$56
#   to a3=$CCA+$866). For each valid region (+$18) whose grid key +$52/+$54 brackets the
#   marble tile, calls region_update $F96A to (de)activate it and run its enter/exit
#   logic (sets +$18/+$1A, plays edge sounds, handles the goal/fall). This is what makes
#   exactly one region "active" for surface_interaction $16900 to read.
00F96A  region_update    enter/leave handler for one region struct (a5). On enter sets
#   +$18 valid / +$1A=3 / copies +$3A; on leave clears +$18/+$1A, decrements counters,
#   and dispatches the region's terrain code +$1F (calls $11C24 for the transition).
01470A  surface_state    per-object surface-state update (a0=obj): copies the active
#   region's slope code into obj+$62, plays the edge sound for codes $10/$32-$37, and
#   sets the obj contact/anim state (+$60/+$64/+$69/+$6A) from $14ED8. Called by the
#   slope/edge cases to latch the marble onto a region.
# The full marble per-frame velocity chain, traced (no assumptions):
#   $013C38 save_prev_state (stores prev tile pos obj+$32/$34, prev z obj+$2A, prev
#     terrain obj+$1D) -> NOT velocity.
#   marble_input $12F8C  -> input ADD to velocity ($0130DA/$0130E0), then
#   $14E7E -> $14D28 speed_cap_and_roll: computes speed (|vY|+3/8*|vX|), advances the
#     roll anim/sound (obj+$D6, table $201A, $A5F0), and CLAMPS velocity to a max
#     speed ($40000/$50000) by scaling vX/vY at $014E62/$014E6E -> friction, NOT slope.
#   surface build: course_init/$E6B2 -> build_surface $DEBC -> sub_00E158 -> $CCA grid.
016900  surface_interaction  THE surface->velocity step (a5=marble). LOOPS over all 25
#   region structs (a4=$CCA, += $56; loop $016986 counter -$5 vs $19=25; $01802A advances,
#   $016990 skips regions with $18(a4)==0). For each VALID region: d5 = $C(a4)refX - $6C4
#   marbleX, d4 = $10(a4)refY - $6C6 marbleY (vector marble -> this region's ref point);
#   d0 = $1F(a4) terrain code (5..59); JMP $2(pc,d0.l) via the jump table at $016A00 (55
#   BRA.w entries, d0=(code-5)*4, target=entry+2+off). So every region the marble overlaps
#   contributes; a region's handler range-checks (d5,d4) so it only acts within its bounds.
#   LOOP TAIL $01807C: if wall flag $6A1 set -> marble posX($C(a5))=$6B8, NEG.l (a5) vX;
#   if $6A2 -> posY($10(a5))=$6BC, NEG.l $4(a5) vY = the wall BOUNCE (clamp + reverse).
# THE SLOPE FORCE (proven), code 11 case $17AA8 (and 13 -> $17C88): a dead-zone box test
#   ($17AE8) leaves the marble at rest when centred; otherwise normalise (d5,d4) via
#   JSR $23EB4 (divide by octagonal distance), ASL #2 (x4) and ADD to velocity:
#   $017BDE ADD.l d0,(a5) -> vX += ux*4 ;  $017BE8 ADD.l d0,$4(a5) -> vY += uy*4.
#   PROVEN end to end: $9A6 region record -> $CCA struct ref point -> $017BDE/$017BE8 vel.
# THE $016A00 JUMP TABLE, all 55 targets decoded (terrain code -> behaviour):
#   FLAT/no-force ($18024): codes 6-9,14,15,17,25,40-44.
#   SLOPE (vXY += unit(ref-pos)*4): 11 ($17AA8), 13 ($17C88).  FUNNEL (set vel): 12 ($178A6).
#   PROXIMITY TRIGGER (within dist $38 of ref -> set $6A1/$6A2 + sound $21ADC): 5,26-31
#     ($16C0C..); directional sound-only variants: 16,50-55.
#   WALL/EDGE -> shared $180AC: 21,22,33,35,36,38,39.
#   HARD EDGE/FALL/GOAL (set/clr vel + $E872/$EA10/$15FC8/$FD68): 10,18,19,20,32,34,37.
#   geometric/border variants (fall to flat): 23,24,45-49,56-59.
#   Friction (ice/grating) is NOT here: it is the per-region max-speed selector on the
#   marble +$1A read by $14E7E (picks $40000 vs $50000) in the $14D28 speed cap.
# THE $CCA SURFACE-REGION STRUCTS: 25 structs, $56 (86) bytes each ($CCA..$CCA+$866;
#   activator loop $12B02 and producer $F8FC/$F96A both stride $56). Fields: +$C/+$10 =
#   downhill reference point (refX,refY); +$18 valid; +$1A contact/anim state; +$1E
#   counter; +$1F terrain code (5..59, the jump index); +$19/+$1B neighbour/transition
#   code; +$52/+$54 grid key (col,row) matched vs the marble tile to pick the ACTIVE
#   region ($F906 loop); +$36 = slope live target, refreshed from the marble obj $FD2C
#   ($54/$58) for codes 11/13 at $12B42/$12B58 (keeps the slope pushing as it moves).
00E944  iso_to_cell      take the iso-projected SCREEN position ($6C4,$6C6) and derive the cell coordinates the region map / terrain_lookup use: $6CA = $6C4>>3, $6CC = $6C6>>3 (8-px iso cell), $6D2 = $6C4&7, $6D4 = $6C6&7 (sub-cell), $6D6 = 1 if $6D2<=$6D4 else 0 (which DIAGONAL HALF of the cell -> matches the iso diamond / ramp-edge split). So the region records are in iso-screen-cell space; each record's |byte0|,|byte1| is a start cell and the sign of byte0/byte1 is its diagonal direction. For practy all 10 boundaries are PARALLEL diagonals = a ladder of terraces; $FF-sided ones are the fall-off edges. OPEN: exact projection scale/offset to overlay them on the course art pixel-accurately.
019390  trackball_decode read the mouse/TRACKBALL via JOY0DAT/JOY1DAT ($DFF00A/$DFF00C), per player (d5); decode the Amiga quadrature (EOR adjacent counter bits for direction, +/-$20 per step) into an X/Y delta. (CIA fire buttons read at $BFE001 / sub_018F90.) The delta is the marble's roll force.
0192EC  control_update   per-player loop: trackball_decode then apply each player's delta to the marble via $18FCC, accumulating/scaling it into the roll-force accumulators $195EE/$195EA. The marble is a 3-D point mass: position (x,y,z) at obj+$C/+$10/+$14, velocity (vx,vy,vz) at obj+0/+4/+8, integrated each frame by object_draw $14EF0 (pos += velocity) then iso-projected by set_draw_pos $E872.
# THE per-region SLOPE field is the COURSE DESCRIPTOR at Track header +0 ($9A6). It is
# a list of 8-byte REGION RECORDS (origin x/y, size x/y, base height, 5-bit edge-shape,
# 3-bit slope direction -> $2504 8-dir vector); build_surface $DEBC / build_region
# $E158 expand them into the 25 $CCA region structs at load time, and surface_interaction
# $16900 reads those structs each frame (above). (Superseded earlier guess: there is no
# $342/$5DE/$63C parallel-array / 7x7-prototype scheme -- that was a data-shape guess
# the velocity trace disproved.) The PLACEMENT TABLE $129FC->$91C places visible objects
# only; terrain_lookup $12B9E/$9D4 is a coarse zone/edge classifier, not the slope field.
012B9E  terrain_lookup   a COARSE zone/edge lookup (NOT the per-location slope field). Walks the small Track block at header+8 ($12F74 -> $9D4): 5-byte records [+0 dx signed][+1 dy signed][+2 length][+3 type_a][+4 type_b], $FFFF-terminated (practy has only 10) = a 2-D diagonal-boundary partition classifying a few large zones and the off-course edge. Tests the marble's iso-cell ($6CA/$6CC + $6CE/$6D0) vs each boundary, writing a type to +$1B; type $FF ($012D44) = off-course -> fall/death ($5E4). OPEN: the type->slope-acceleration table.

# --- the object / actor (sprite-animation) system (Part V) ---
# The .ilb/.vlb cells are drawn as BLITTER objects (no hardware sprites: no
# $DFF0A0-$DFF0DC anywhere). The playfield is 4 bitplanes, so each on-screen object
# is 16-colour = 4 planes. A 4-plane object is built from TWO consecutive 2-plane
# descriptor cells (e.g. the goal flag = cells 0+1; the marble = an image pair),
# while the composited scenery cells (+0=00) already hold 4 planes in one cell.
011D1C  blit_object      blit one object into the 4-plane playfield. Arg $8(a6) = a cell descriptor (the unpacked-.ilb 20-byte record); $E/$12/$16/$1A = x/screen-offset/word-x/y. Reads $2(a0)=width words, $6(a2)=one-plane size, $8(a0)=source. Loops over the screen bitmap's plane-pointer table at $7B30+8 for FOUR planes ($011E4C ADDA.w #$10 = 4 longs), advancing the source by one-plane-size each plane -> reads 4 sequential one-plane blocks from the cell source = 2 descriptor cells. Optional cookie-cut mask path when desc+1 bit0 set and desc+$C (mask) != 0 (minterm $FCA), else plain copy (minterm $5CC).
0104C4  draw_object_wrap draw an object at (x,y) with vertical wraparound: clips to the $200 (512) tall scroll buffer and calls blit_object once or twice (top/bottom halves) so objects straddling the scroll seam render whole. Reads scroll offset via $7B28.
01D3B2  actor_update     the per-frame actor update+draw loop. Walks an array of 20-byte actor records (stride $14): +0 = pointer to the current sprite CELL to draw (passed to draw_object_wrap), +4 = animation-script pointer (8-byte entries: cell ptr + timing; advanced when the +8 frame timer expires, $FFFFFFFF-terminated -> loop), +8 = frame timer, +A/+C = x/y position (y biased by scroll $D30C), +E/+10 = anim/wrap state. Pulls anim scripts from the per-course Track blob via $FD2C (e.g. $280(a0)/$284(a0)); per-actor sub-records live at $18A6 (stride $10). Frame durations randomised via rng $8F96.

# --- the per-course object PLACEMENT table (inside the Track segment) ---
# Track header field +4 -> $129FC = a small struct {+0 = main placement list,
# +4/+8 = a secondary list}. The placement list is an array of 3-BYTE RECORDS
# [X][Y][type] terminated by a leading $FF. The engine places object kind `type`
# (0..7) at the isometric grid cell (X,Y); the screen seed is X*8+4 / Y*8+4
# ($6C4/$6C6) before the isometric transform ($6718). Decoded per course by
# extract/cmd/tracks: practy 59, beginr 79, interm 87, aerial 159, silly 104,
# ultima 144 objects.
000690  g_placement      (global) the Track placement struct pointer (= $129FC); +0 = the [X][Y][type] object-placement list, $FF-terminated.
0122AC  place_objects    walk the placement list filtered by type byte (+2) == the request's $1B (proximity/picking helper). The placement `type` (0..7) is the marble's TERRAIN/OBSTACLE interaction kind (collision query $012600 finds the nearest record within a radius, isometric-transforms it $6718, and stores type->$6A0 = marble +$1B), NOT a free sprite index — static scenery is in the .mlb tilemap. The marble's reaction per type is the physics dispatch on $6A0 (Part V §3, TODO).
