# --- the sprite/level-bank loader and codec (Part IV §3 tile map, §4 obstacles) ---
0080B4  ilb_load         load an .ilb/.vlb sprite bank for the current course: index a buffer table, read+unpack the WHOLE file (bank_read_unpack — one ByteRun1 stream), then walk the bank's descriptors IN the unpacked buffer at buf+2, stride $14=20. Each 20-byte descriptor: +0 type byte (1=stored cell, 0=composited), +1 flag byte (bit0 = process), +2 width in 16-px words, +4 height in rows, +6 one-plane byte size (=w*2*h), +8 source offset (relocated +buf), +$C dest cell ptr (allocated from buffer slack or relocated), +$10 aux ptr (relocated). Cells' source pixel data is contiguous after the descriptor array; plane count = source span / +6 (2 for small objects, 4 for scenery). Composites type-0 cells via composite_planes.
007F38  mlb_load         load an .mlb level module (different container: flag $01 + a 5-entry offset-table header) through the SAME bank_read_unpack path, so it shares the ByteRun1 codec.
007E4E  bank_read_unpack file loader: dos_open(name, MODE_OLDFILE) -> dos_read into the buffer -> dos_close; then, if the global flag $8351 is set, move the loaded (packed) data to the high end of the buffer and ByteRun1-decompress it in place (unpack_byterun1) back to the start. Returns the unpacked length.
007DA6  pool_alloc       bump allocator: hand out $8(a6) bytes from the work pool tracked by $8340/$8344 (high-water at $8334), growing it via $8D74 when exhausted.
009118  unpack_byterun1  THE BITMAP CODEC = ByteRun1 / PackBits (identical to IFF ILBM bodies). unpack(src@8, dst@C, srclen@10, dstlen@14): read a signed control byte n — n in 0..127 copies the next n+1 bytes literally; n in -1..-127 repeats the next single byte (1-n) times (2..128); n = -128 is a no-op. Stops at dstlen or end of source.
008026  composite_planes one bank cell: zero the dest cell (size = desc->4 * desc->2 * 2), then OR four consecutive planar source blocks (desc->8) into it — builds the final 2-plane (16-wide) cell from the unpacked source.

# --- display + palette ---
0248F0  gfx_setrgb4      graphics.library SetRGB4 wrapper (-$120 via the graphics base at $1F4); applies colours into a ViewPort.
01F5CC  apply_palette    sets the playfield ViewPort's colours (calls gfx_setrgb4 with the ViewPort at $1FA82). The colour VALUES come from the loaded course .mlb, not from the .dat — the program holds no colour table.
009910  mlb_draw_column  draw a column/region of the course from the tilemap: reads the tilemap (buffer+$12 via $83E4) and the 4 tile-graphic plane pointers, indexing tiles by (index*36); calls draw_tile per tilemap entry.
0099C0  draw_tile        blit one 8x8 4-plane tile into the screen bitmap: tile source = plane[(index>>1)*16 + (index&1)] (even/odd tiles byte-interleaved); copies 8 rows (1 byte each, at even source offsets 0,2,..,$E) for each of the 4 planes to the destination at row stride $24=36. So .mlb tiles are 8x8, 16-colour.
# The per-course palette lives in each course's .mlb level module at offset 0x17:
# 16 big-endian $0RGB words (playfield colours 0-15; 0-6 = shared grey scenery
# ramp, 7-15 = per-course object colours). Confirmed by reading $DFF180 live under
# FS-UAE (= practy.mlb+0x17 byte-for-byte). The engine also programs the blitter
# ($DFF040-$DFF066) and DMACON ($DFF096) directly and reads JOYxDAT for input.

# --- top level: the game shell and the real-time "Framer" main loop ---
# main ($243E8, the C entry's argv setup at cstart) calls game ($2B68).
002B68  game_shell       opens intuition.library, runs init/draw (sub_00CA14, which reaches the render chain) and start_gameplay ($6FC6), then an IDCMP message loop: GetMsg ($249B4) / ReplyMsg ($249C8), dispatch on the message Class field (+$14 == 8/9/10) to the start-course handlers ($3410/$3432) or quit. This is the menu/window front-end, NOT the per-frame loop.
006FC6  start_gameplay   thin wrapper over $6F20(flag); $6FDC creates the real-time task.
006FDC  create_framer    create an exec task named "Framer" (name string at $70D8) via $248D4, entry = framer_task $6DA8. Wires up its frame-rate-tracking globals ($7060/$7092/$7096 handler ptrs, $70B8 frame-time histogram).
006DA8  framer_task      THE real-time main loop (own task). Each iteration: Wait ($24B54) for the vblank signal, then call frame_tick $7596(frame_index); meanwhile it tallies how many vblanks each frame took into the $70B8 histogram (the "Framer" = frame-rate meter) and polls GetMsg ($249B4) for a quit/stop message to exit.
007596  frame_tick       one frame of work, dispatched by flags: per-subsystem gates $5BCA, $1E20D, and the gameplay flag $5DB; advances the frame counter $230 and latches the scroll position ($D30C -> $7692). The playing branch runs the game logic + render (the sub_00CA14/sub_014EF0 chain). Returns whether a redraw is needed ($86B4).

# --- the per-frame render pipeline (reached from frame_tick's gameplay branch) ---
00E872  set_draw_pos     set the screen draw position for an object: copy its world pos (obj+$C/+$10/+$14) to $6B8/$6BC/$6C0, the >>$13 tile coords to $6CE/$6D0, then call $E944 to project to the iso screen seed $6C4/$6C6/$6C8. Called before each object draw.
014EF0  object_draw       per-object update+draw: advance the object's world pos by its velocity (obj+$C += obj+0, +$10 += +4, +$14 += +8), set_draw_pos, then draw the object plus up to three actor passes (a main sprite + shadow/secondary passes) chosen by the object subtype obj+$1B (==1/2/3), each at its own Y offset ($1E8/$238/$210) via actor_update; a per-frame "already drawn" bitmask ($5F6) prevents double-draws.
# Render leaf routines (annotated elsewhere): mlb_draw_column $9910 + draw_tile $99C0
# draw the .mlb tilemap background; place_objects $0122AC walks the Track placement
# list; actor_update $01D3B2 -> draw_object_wrap $0104C4 -> blit_object $011D1C draw
# the sprites. The .ilb cell ENCODING is selected by the descriptor type byte (+0):
# type-0 (+0=00) = composited scenery, span=4*cz, 4 plane-blocks (mask built by
# composite_planes $8026); type-1 (+0=01) = free sprites, span=2*cz, row-interleaved
# 4-plane at half height (the flag/marble/creatures).

# --- the object / actor (sprite-animation) system (Part V) ---
# The .ilb/.vlb cells are drawn as BLITTER objects (no hardware sprites: no
# $DFF0A0-$DFF0DC anywhere). The playfield is 4 bitplanes, so each on-screen object
# is 16-colour = 4 planes. A 4-plane object is built from TWO consecutive 2-plane
# descriptor cells (e.g. the goal flag = cells 0+1; the marble = an image pair),
# while the composited scenery cells (+0=00) already hold 4 planes in one cell.
011D1C  blit_object      blit one object into the 4-plane playfield. Arg $8(a6) = a cell descriptor (the unpacked-.ilb 20-byte record); $E/$12/$16/$1A = x/screen-offset/word-x/y. Reads $2(a0)=width words, $6(a2)=one-plane size, $8(a0)=source. Loops over the screen bitmap's plane-pointer table at $7B30+8 for FOUR planes ($011E4C ADDA.w #$10 = 4 longs), advancing the source by one-plane-size each plane -> reads 4 sequential one-plane blocks from the cell source = 2 descriptor cells. Optional cookie-cut mask path when desc+1 bit0 set and desc+$C (mask) != 0 (minterm $FCA), else plain copy (minterm $5CC).
0104C4  draw_object_wrap draw an object at (x,y) with vertical wraparound: clips to the $200 (512) tall scroll buffer and calls blit_object once or twice (top/bottom halves) so objects straddling the scroll seam render whole. Reads scroll offset via $7B28.
01D3B2  actor_update     the per-frame actor update+draw loop. Walks an array of 20-byte actor records (stride $14): +0 = pointer to the current sprite CELL to draw (passed to draw_object_wrap), +4 = animation-script pointer (8-byte entries: cell ptr + timing; advanced when the +8 frame timer expires, $FFFFFFFF-terminated -> loop), +8 = frame timer, +A/+C = x/y position (y biased by scroll $D30C), +E/+10 = anim/wrap state. Pulls anim scripts from the per-course Track blob via $FD2C (e.g. $280(a0)/$284(a0)); per-actor sub-records live at $18A6 (stride $10). Frame durations randomised via rng $8F96.

# --- per-course course/level setup (loads the Track "level layout" file) ---
003286  course_init      (re)initialise the current course: free old per-course data (sub_003204), then mlb_load the course .mlb (t_mlb_names $3496) and load the rest of the per-course assets, all indexed by g_course $5D6.
003176  load_track_data  load the per-course Track file: index t_track_names $353C by g_course, LoadSeg it via $8CF0, store the seglist at $3554, then distribute the loaded segment's header pointers (10 longs at seg+0..+$24) to the actor-system globals: +0->$9A6, +4->$129FC, +8->$12F74, +$C->$1ED44, +$10->$89C2, +$14->$FD2C (animation scripts), +$18->$19CD0, +$1C->$1ABE0, +$20->$1BE48, +$24->$1D334.
003204  free_track_data  free the Track seglist ($3554) and null all the actor-system globals it populated.
0353C   t_track_names    table of 6 pointers (indexed by g_course) to the Track filenames at $3506 ("PrcTrack","BegTrack","IntTrack","AerTrack","SilTrack","UltTrack"). NB: *Track is the per-course LEVEL/OBJECT LAYOUT data (actor placements + animation scripts), NOT music — music/SFX is the separate *Snd files.
03554   g_trackseg       (global) seglist BPTR of the currently loaded course Track file (the per-course object/actor data blob).
00FD2C  g_animscripts    (global) pointer into the loaded Track segment to its animation-script table (= Track seg header field +$14); the actor loop reads cell-animation scripts from here.
008CF0  loadseg_file     LoadSeg(name) wrapper (dos.library -$96 via $24814) + return the first seg's data pointer (seglist walk $8CC0). Used to load the Track files.

# --- the per-course object PLACEMENT table (inside the Track segment) ---
# Track header field +4 -> $129FC = a small struct {+0 = main placement list,
# +4/+8 = a secondary list}. The placement list is an array of 3-BYTE RECORDS
# [X][Y][type] terminated by a leading $FF. The engine places object kind `type`
# (0..7) at the isometric grid cell (X,Y); the screen seed is X*8+4 / Y*8+4
# ($6C4/$6C6) before the isometric transform ($6718). Decoded per course by
# extract/cmd/tracks: practy 59, beginr 79, interm 87, aerial 159, silly 104,
# ultima 144 objects.
000690  g_placement      (global) the Track placement struct pointer (= $129FC); +0 = the [X][Y][type] object-placement list, $FF-terminated.
0122AC  place_objects    walk the placement list filtered by type byte (+2) == the request's $1B (proximity/picking helper). The placement `type` (0..7) is the marble's TERRAIN/OBSTACLE interaction kind (collision query $012600 finds the nearest record within a radius, isometric-transforms it $6718, and stores type->$6A0 = marble +$1B), NOT a free sprite index — static scenery is in the .mlb tilemap. The marble's reaction per type is the physics dispatch on $6A0 (Part V §3, TODO).
01D3EC  actor_setup      sets up the dynamic actors (enemies / animated obstacles): the 3-record actor array at $FD2C+$23C (stride $14, end +$3C); pulls templates/anim scripts from the Track segment (anim table at $FD2C+$280). Course-gated ($5D6==3 path tests $78E). The type(0..7)->object/anim definition tables are the other Track header pointers ($1ED44 ptr array @+$C, $89C2 @+$10, anim $FD2C @+$14) + a per-course copper-style colour table inside the +$10 block ($DFF180..$DFF192) — decode TODO.
