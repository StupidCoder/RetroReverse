Stunt Car Racer (Amiga) — track viewer

drag to rotate · scroll to zoom

Each of the eight league circuits is the game's own track data, decoded purely in Go from the disk image — no emulation. The disk has no filesystem; a table of handles at $1F0A2 locates each track's byte stream, which is a run-length-encoded list of sections that the loader ($5AE46) walks to lay out the circuit. This viewer re-runs that walk (package track) to place each section node, then draws the track as a hidden-line wireframe ribbon — the two rails plus a rung per section, coloured along the lap (blue → red) with the green marker at the start/finish line. The section decode (types, parameters, segment lengths) and these node arrays are verified coordinate-exact against the original loader executed on our Go 68000 core, for all eight tracks.

Work in progress: these node arrays are the loader's internal section layout, not yet the final 3-D world centre-line, so each circuit currently reads as a flattened plan. The renderer composes the visible track from these plus a per-section heading and elevation (the ramps, jumps and famous over-passes); recovering that world transform is the active next step, after which the tracks will stand up properly in 3-D. See the write-up (Part IV) for how the track format was decoded.
